## 1. Adaptive Routing Decision Tree

Every cognitive task generated by CENSA goes through our multi-level adaptive routing model:

```
                      ┌────────────────────────┐
                      │    Task Generated      │
                      └───────────┬────────────┘
                                  │
                       [Complexity Check]
                                  │
                  ┌───────────────┴───────────────┐
                  ▼                               ▼
             [Complexity = Simple]           [Complexity = Complex]
                  │                               │
            (Route to Local)              (Route to Fireworks)
                  │                               │
           ┌──────┴──────┐                 ┌──────┴──────┐
           ▼             ▼                 ▼             ▼
       [Ollama]      [AMD Instinct]     [Fireworks]  [vLLM Fallback]
```

---

## 2. Decision Factors

1.  **Complexity**:
    - *Simple* (Intent parsing, summary logs) ➔ Routed to Local Gemma-2-9b-it / Ollama.
    - *Complex* (Task DAG code compiles, sandboxed VM test runs) ➔ Routed to Fireworks Llama-3-70b-Instruct.
2.  **Reasoning Scale**: Large multi-stage tasks route to 70B parameter models on Fireworks, preserving local memory bandwidth.
3.  **Token Budget**: Automatic compression via pxpipe reduces token footprints by ~52% before final routing.
4.  **Provider Availability**: Failovers trigger automatically in under 500ms on connection errors, falling back to deterministic local mock routines to protect the session state.
