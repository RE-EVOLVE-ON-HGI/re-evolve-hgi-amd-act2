# Live Demo Walkthrough Script
## Re-Evolve on HGI · v2.0.0-final · AMD Developer Hackathon ACT II

---

## 1. Demo Flow Steps

This guide outlines the step-by-step walkthrough of the Re-Evolve live demonstration.

### Step 1: Open the Landing Page Presentation
*   **Action**: Navigate to `https://frontend-alpha-rose-25.vercel.app`.
*   **Visual**: The visitor is greeted by a dark, starry cinematic landing page displaying the neural connection orb.
*   **Narrative**: Explain the core vision: Re-Evolve is the world's first Human-Governed Adaptive Intelligence (HGI) Operating System built specifically to coordinate agent swarms safely on high-throughput AMD Instinct compute grids.

### Step 2: Scroll Through the Keynote Chapters
*   **Action**: Scroll down the page to review the storytelling structure:
    *   *Chapter 2 & 3*: The Crisis (fragmented agents) vs The Convergence.
    *   *Chapter 4*: CENSA Cognitive Orchestration (Intent Parsing & Task DAG generation).
    *   *Chapter 5*: Panani X Secure Isolates (Node VM Sandboxing).
    *   *Chapter 6*: Memory Vault Persistent Graph (pgvector & Qdrant integration).
    *   *Chapter 7*: Kavacha Zero-Trust Shield (Safety guards & cost audit logging).
    *   *Chapter 8*: AMD AI Fabric (ROCm accelerators & Fireworks AI failovers).

### Step 3: Run the Guided Intelligence Simulator
*   **Action**: Scroll to **Chapter 9 (Mission Builder & Judge Mode)**:
    1. Click on one of the sample objectives: *"Optimize supply chain routing"*.
    2. Click **Initialize & Execute Simulation**.
*   **Visual**: The progress bar updates, the terminal logs feed outputs, and the Live Status Checklist lights up green checkboxes for Intent, Planning, Model Selection, Execution, Governance, and Memory.
*   **Narrative**: Point out how CENSA automatically decomposes the target objective into a task graph, allocates specialist agents inside secure sandboxes, retrieves cross-session memory indexes, and audits all actions via Kavacha before producing the final response.

### Step 4: Access Gated Enterprise Modules
*   **Action**: Scroll back to the top navigation header and click **Enter Workspace**:
    1. When the passcode modal appears, type: `AMD-GOLD`.
    2. Click **Decrypt & Enter**.
*   **Visual**: The system redirects to the live developer dashboard `/hq` containing the telemetry charts, agent registries, and workflow visualizers.
*   **Narrative**: Demonstrate how enterprise administrators audit running agent containers, track active AMD hardware configurations, and check compliance rule sheets.

### Step 5: Test Gated Navigation Boundaries
*   **Action**: In the left sidebar of the dashboard, click on one of the gated enterprise menu items (e.g. *Alpha Omega Chamber* or *Agent Council*).
*   **Visual**: The system blocks the route, intercepting the 404 and displaying the custom warning page: **"Enterprise Platform Modules"**.
*   **Narrative**: Explain that advanced enterprise modules are gated during the hackathon demonstration to focus evaluation on the core operating system kernels.
*   **Action**: Click **Return to Hackathon Experience** to return to the landing presentation.
